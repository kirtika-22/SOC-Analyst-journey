# What are Windows Event Logs?

Windows Event Logs are built-in logs that record system, security, and application activities on Windows systems.

They help SOC analysts:
- Detect attacks
- Investigate incidents
- Troubleshoot system issues
- Monitor user activity
- Perform digital forensics

---

# Types of Windows Logs

## 1. Security Logs
Contains authentication, authorization, account management, privilege changes, and audit events.

Examples:
- Login
- Logoff
- Failed login
- Account creation
- Password reset

---

## 2. System Logs

Stores events generated by Windows components.

Examples:
- Driver failures
- Service crashes
- System startup/shutdown

---

## 3. Application Logs

Generated by installed applications.

Examples:
- SQL Server
- IIS
- Antivirus
- Microsoft Office

---

# Why Security Logs are Important for SOC

Security logs help analysts:

- Detect brute-force attacks
- Detect privilege escalation
- Track user activities
- Detect lateral movement
- Investigate compromised accounts
- Perform incident response

# Important Windows Event IDs

## Event ID 4624
### Successful Logon

Purpose:
Successful authentication.

SOC Importance:
- Track user access
- Detect unusual login locations
- Verify normal login behaviour

Detection:
- Login from foreign country
- Impossible travel
- Unusual time

Mitigation:
- MFA
- VPN
- Strong password policy

---

## Event ID 4625
### Failed Logon

Purpose:
Failed authentication attempt.

SOC Importance:
- Brute-force detection
- Password spraying
- Credential stuffing

Detection:
- Multiple failures
- Same IP attacking many accounts

Mitigation:
- Account lockout
- MFA
- Strong passwords

---

## Event ID 4648
### Logon using Explicit Credentials

Purpose:
Credentials supplied manually.

SOC Importance:
- Detect lateral movement
- Detect Pass-the-Hash activity
- Detect suspicious credential usage

Detection:
- Unexpected account usage
- Non-business hours

Mitigation:
- Least privilege
- Restrict credential usage

---

## Event ID 4672
### Special Privileges Assigned

Purpose:
Administrative privileges assigned during login.

SOC Importance:
- Detect privilege escalation
- Detect compromised admin accounts

Detection:
- Admin privileges assigned unexpectedly

Mitigation:
- Least privilege
- Regular privilege review

---

## Event ID 4688
### New Process Created

Purpose:
A process starts.

SOC Importance:
- Detect malware
- Detect PowerShell attacks
- Detect suspicious executables

Detection:
- Unknown executable
- Temporary folder execution

Mitigation:
- EDR
- Application Whitelisting

---

## Event ID 4689

Purpose:

Mitigation:
