# Day 10 - Windows Internals for SOC Analysts

## Objective

To understand the internal architecture of the Windows Operating System and learn how SOC Analysts investigate Windows systems using processes, event logs, system utilities, command-line tools, and forensic utilities.

---

# Windows Operating System Architecture

Windows is divided into two execution modes:

## 1. User Mode

User Mode is where normal applications execute.

### Examples
- Google Chrome
- Microsoft Edge
- Notepad
- Microsoft Word
- Text Editors
- Browsers

### Characteristics
- Cannot directly access hardware.
- Requests system resources through system calls.
- More secure because application crashes do not directly crash Windows.

---

## 2. Kernel Mode

Kernel Mode has unrestricted access to system resources.

### Responsibilities
- Memory Management
- Process Scheduling
- Device Management
- Hardware Communication
- Interrupt Handling

If a kernel component crashes, the entire operating system may crash (Blue Screen of Death).

---

# Key Windows Components

## 1. Hardware Abstraction Layer (HAL)

- Acts as a bridge between hardware and Windows.
- Allows Windows to work on different hardware platforms.
- Hides hardware-specific implementation from the operating system.

---

## 2. NTOSKRNL.EXE

The Windows Kernel.

### Responsibilities
- Memory Management
- Process Management
- Thread Scheduling
- Security
- Interrupt Handling

This is the core of the Windows operating system.

---

## 3. System Services & Drivers

- Kernel-mode drivers communicate with hardware devices.
- Provide services required by Windows.
- Examples:
  - Keyboard Driver
  - Network Driver
  - Storage Driver

---

## 4. Win32 Subsystem

Responsible for Windows graphical interface.

Provides:
- GUI
- Console Applications
- Windows API support

---

# Essential Windows Processes

## 1. System Process

- Core kernel process.
- Manages hardware-related operations.

---

## 2. wininit.exe

Purpose:
- Initializes Windows services after system boot.

---

## 3. csrss.exe

Client/Server Runtime Subsystem.

Responsible for:
- Console windows
- Thread creation
- User sessions

---

## 4. svchost.exe

Service Host Process.

Purpose:
- Runs Windows services.
- Multiple svchost.exe processes are normal.

SOC analysts should investigate unusual or suspicious svchost instances.

# Important Windows Directories

## 1. C:\Windows\System32

Contains:
- Operating system executables
- DLL files
- Windows utilities

Examples:
- cmd.exe
- taskmgr.exe
- regedit.exe

### SOC Importance

Frequently targeted by malware through:
- DLL Injection
- Persistence
- File replacement

---

## 2. C:\Users

Contains individual user profiles.

Each user has folders such as:
- Desktop
- Documents
- Downloads
- Pictures

### SOC Importance

Malware often stores payloads inside user directories because users usually have write permissions.

---

## 3. C:\Program Files

Contains installed applications.

- Program Files → 64-bit applications
- Program Files (x86) → 32-bit applications

### SOC Importance

Monitor for:
- Unauthorized software
- Suspicious installations

---

## 4. AppData

Common paths:

```
%APPDATA%
%LOCALAPPDATA%
```

Example:

```
C:\Users\<Username>\AppData\Roaming
```

Contains:
- User-specific application data
- Configuration files
- Cache

### SOC Importance

Attackers commonly hide malware here for persistence.

# Windows Persistence Techniques

## 1. Startup Folder Abuse

Location:

```
C:\Users\<Username>\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup
```

Attackers place:
- .exe
- .bat
- Scripts

These execute automatically when the user logs in.

---

## 2. Registry Run Keys

Example:

```
HKCU\Software\Microsoft\Windows\CurrentVersion\Run
```

Attackers add malicious programs here so they automatically execute during login.

- Detect persistence mechanisms
- Analyze suspicious processes
- Review Windows Event Logs
- Hunt for Indicators of Compromise (IOCs)
- Perform effective incident response
- Understand attacker behavior inside Windows systems
