# Day 11 - Interview Questions with Answers (Logs & Log Management)

## 1. What is a log?
**Answer:**  
A log is a record of events generated by an operating system, application, network device, or security tool. It helps monitor activities and investigate security incidents.

---

## 2. Why are logs important in cybersecurity?
**Answer:**  
Logs help detect attacks, investigate incidents, monitor user activity, perform forensic analysis, and meet compliance requirements.

---

## 3. What are the different types of logs?
**Answer:**
- System Logs
- Security Logs
- Application Logs
- Firewall Logs
- Web Server Logs
- Authentication Logs
- DNS Logs
- Email Logs
- Proxy Logs
- Cloud Logs

---

## 4. What is the difference between system logs and security logs?
**Answer:**
- **System Logs:** Record OS events like boot, shutdown, and hardware issues.
- **Security Logs:** Record login attempts, malware detection, and security events.

---

## 5. What is log management?
**Answer:**  
Log management is the process of collecting, storing, monitoring, analyzing, and protecting logs from different sources.

---

## 6. Explain the log lifecycle.
**Answer:**
1. Log Generation
2. Log Collection
3. Log Forwarding
4. Log Normalization
5. Analysis & Correlation
6. Storage & Archiving

---

## 7. What is log normalization?
**Answer:**  
Log normalization converts logs from different formats into a standard format so they can be analyzed easily.

---

## 8. What is log correlation?
**Answer:**  
Log correlation combines logs from multiple sources to detect suspicious activities or attacks.

---

## 9. What are common log formats?
**Answer:**
- Syslog
- JSON
- CEF
- LEEF
- Plain Text/CSV

---

## 10. What is Syslog?
**Answer:**  
Syslog is a standard logging format used by Linux and Unix systems.

---

## 11. What is JSON logging?
**Answer:**  
JSON logging stores logs in a structured key-value format, making them easy to search and analyze.

---

## 12. What is CEF?
**Answer:**  
CEF (Common Event Format) is a standard log format developed by ArcSight for security devices.

---

## 13. What is LEEF?
**Answer:**  
LEEF (Log Event Extended Format) is a log format developed by IBM QRadar.

---

## 14. What is log retention?
**Answer:**  
Log retention is the practice of storing logs for a specific period for security, compliance, and investigations.

---

## 15. What is SIEM?
**Answer:**  
SIEM (Security Information and Event Management) collects, correlates, and analyzes logs from multiple sources to detect threats.

---

## 16. Why do SOC analysts rely on logs?
**Answer:**  
SOC analysts use logs to detect attacks, investigate incidents, perform threat hunting, and monitor systems.

---

## 17. What are common log sources?
**Answer:**
- Operating Systems
- Firewalls
- Applications
- Databases
- Web Servers
- DNS Servers
- Email Gateways
- Cloud Platforms

---

## 18. What information do firewall logs provide?
**Answer:**  
Firewall logs show allowed and blocked traffic, source/destination IPs, ports, and protocols.

---

## 19. What are DNS logs used for?
**Answer:**  
DNS logs record domain lookups and help detect phishing or malicious domains.

---

## 20. What are authentication logs?
**Answer:**  
Authentication logs record successful and failed login attempts.

---

## 21. What information is available in web server logs?
**Answer:**  
They record client IPs, requested URLs, HTTP methods, response codes, and timestamps.

---

## 22. What are proxy logs used for?
**Answer:**  
Proxy logs monitor users' internet activity and help identify access to malicious websites.

---

## 23. What are cloud logs?
**Answer:**  
Cloud logs record user activities, API calls, and resource usage in cloud platforms like AWS and Azure.

---

## 24. What is Loggly?
**Answer:**  
Loggly is a cloud-based log management platform by SolarWinds used to collect, search, monitor, and analyze logs.

---

## 25. Name some log management tools.
**Answer:**
- Splunk
- ELK Stack
- Graylog
- Wazuh
- IBM QRadar
- Loggly

---

## 26. How do you view logs in Windows?
**Answer:**  
Using **Event Viewer (eventvwr.msc)**.

---

## 27. How do you view logs in Linux?
**Answer:**  
Using **journalctl** and log files stored in **/var/log/**.

---

## 28. What are common logging mistakes?
**Answer:**
- Logs not enabled
- Short retention period
- No filtering
- Ignoring important events
- Logs not protected from tampering

---

## 29. How do logs help during incident response?
**Answer:**  
Logs help identify the attack timeline, affected systems, attacker actions, and evidence for investigation.

---

## 30. Which log is most useful for investigating failed login attempts?
**Answer:**  
Authentication logs and Security logs.
