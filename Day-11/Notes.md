## Objective

To understand what logs are, why they are important in cybersecurity, different log formats, the log lifecycle, common log sources, log management tools, and how SOC analysts use logs to detect and investigate security incidents.

# What are Logs?

Logs are records of events generated by operating systems, applications, network devices, and security tools.

They help security teams:

- Monitor system activity
- Detect suspicious behavior
- Perform forensic investigations
- Meet compliance requirements

---

# Importance of Logs in Cybersecurity

- Detect unauthorized access attempts
- Identify suspicious user activities
- Support threat hunting
- Help investigate security incidents
- Meet compliance and auditing requirements
- Enable SIEM correlation across multiple log sources

---

# Common Log Formats

## 1. Syslog
- Standard logging format for Linux/Unix systems.

## 2. JSON
- Structured key-value format widely used by modern applications and ELK Stack.

## 3. CEF (Common Event Format)
- Standard log format developed by ArcSight.

## 4. LEEF (Log Event Extended Format)
- IBM QRadar log format.

## 5. Plain Text / CSV
- Human-readable log files stored as text or comma-separated values.

# Types of Logs

## System Logs
Record operating system events like boot, shutdown, drivers, and hardware issues.

## Security Logs
Record login attempts, malware detection, IDS/IPS alerts, and antivirus events.

## Application Logs
Capture application errors, user activities, and exceptions.

## Firewall Logs
Show allowed and blocked network traffic.

## Web Server Logs
Record client requests, URLs accessed, and HTTP responses.

## Authentication Logs
Track successful and failed logins from Active Directory, LDAP, or RADIUS.

## DNS Logs
Store DNS queries and domain resolution events.

## Email Logs
Monitor email delivery, spam, and phishing attempts.

## Proxy Logs
Record user internet browsing activity.

## Cloud Logs
Monitor cloud resources, API calls, and user actions.

# Common Logging Mistakes

- Logging disabled on systems
- Short log retention period
- Excessive log collection without filtering
- Ignoring critical event IDs
- Logs not protected from tampering
